# General Development Rules

- Match existing project patterns before introducing new ones.
- Prefer small, focused changes; avoid drive-by refactors unrelated to the task.
- Do not expand scope beyond what was asked.
- Use established design patterns (MVC on the backend; component + hooks on the frontend).
- Format Python with `autopep8`; format JS/TS/HTML/CSS with Prettier.
- Never commit secrets, `.env` files, or credentials.
- If a decision is ambiguous and affects architecture, stop and ask rather than guessing.
